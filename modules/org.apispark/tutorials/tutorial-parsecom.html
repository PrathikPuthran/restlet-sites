Exposing custom APIs for Parse.com backends

1. Introduction

This tutorial describes how to create a custom web API that reexposes existing data stored in a Parse backend. Parse.com is a popular Backend as a Service (BaaS) provider that powers connected mobile applications.

APISpark provides an entity store wrapper for existing Parse data that can be reexposed through one or more custom web APIs.

2. Introduction

Let’s get started by configuring a Parse schema named Contact with following fields :

  • contact_id : autogenerated primary key
  • lastName (string) : last name
  • firstName (string) : first name
  • age (integer) : age

2. Configuration

The second step is to plug your API on your existing Parse.com schema. Unfortunately Parse.com doesn’t provide at the moment introspection support of schemas. This means that APISpark can’t provide automatic configuration and you need to manually configure the mapping between APISpark and Parse.com schemas. But no worry, such configuration is actually quite simple.

First, you need to create an entity store with a source of type “Parse”. This is the way to create a Parse.com entity store wrapper.

Then, configure the general hints to access your Parse.com application in the Settings tab (entry ParseAccount), meaning setting the public and private keys previously gotten.

If the entity field names in APISpark exactly match, you simply have to create an entity with same name and same fields within the store:

It’s possible that there is a gap between your entity elements and the schema elements in Parse.com. For this purpose, you can use the Mapping tab of the entity. This allows specifying the mapping between each element (entity name and field names).

1. Using the entity store wrapper

Since such an entity store wrapper is an entity store, you can simply import it from a Web API project and use it to build your API. See this other tutorial for more details on this feature.

Nevertheless you need to deploy your store to make it actually usable at runtime. For this simply use the “Deploy” button in the store header.
